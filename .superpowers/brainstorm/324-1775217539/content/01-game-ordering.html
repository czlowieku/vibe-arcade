Jak gracz zamawia mini-grę od AI?

To jest core mechanic MVP — od tego zależy cały gameplay loop

A

🎰 Slot Machine / Losowe tokeny

Gracz kręci "kołem fortuny" i dostaje losowe parametry: gatunek + mechanika + motyw. Np. "Platformówka + Grawitacja + Kosmos". Może re-rollować za walutę.

Pros

  • Zaskoczenie i replayability
  • Łatwe do zbalansowania
  • Nie wymaga kreatywności gracza

Cons

  • Mniej kontroli gracza
  • Może frustrować złymi combo
B

🃏 Deck Builder / Karty

Gracz zbiera karty (mechanik, motywów, modyfikatorów) i składa z nich "przepis" na grę. Nowe karty zdobywa za granie. Np. karta "Strzelanie" + karta "Neon" + karta "Boss Fight".

Pros

  • Głęboka progresja
  • Strategia w budowaniu decku
  • Gracz czuje się twórcą

Cons

  • Więcej UI do zbudowania
  • Balansowanie kart
C

💬 Free Prompt

Gracz pisze wolny tekst opisujący jaką grę chce. AI interpretuje i generuje. Jak ChatGPT ale w kontekście gry.

Pros

  • Pełna wolność twórcza
  • Najbardziej "vibe" doświadczenie
  • Prosty UI

Cons

  • Nieprzewidywalna jakość
  • Brak struktury progresji
  • Gracz musi być kreatywny
D

🎯 Hybrid: Tokeny + Prompt

Gracz wybiera bazowe tokeny (gatunek, trudność) ale może dodać wolny tekst jako "flavor". Np. wybiera "Platformówka + Łatwa" i dopisuje "ale żeby były koty jako postacie".

Pros

  • Struktura + wolność
  • Łatwiej kontrolować jakość AI output
  • Progresja przez odblokowanie tokenów

Cons

  • Średnio złożony UI
